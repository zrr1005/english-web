# 英语口语练习网站 · 方案

## 产品定位
一个纯前端的英语口语练习工具，支持导入文本/字幕，提供三种练习模式。

---

## 一、核心功能

### 1. 内容导入
- **粘贴文本**：直接粘贴英文段落
- **上传字幕文件**：支持 .srt / .vtt / .txt
- **自动分句**：按标点（. ! ? ）拆分为句子列表
- 导入后存入 localStorage / IndexedDB，离线可用

### 2. 逐句书写（Dictation）
- 播放一句音频（TTS）
- 用户在输入框默写
- 提交后逐词对比原文，高亮错误
- 显示准确率百分比
- 可重听、可跳过、可重试

### 3. 逐句语音输入（Speaking）
- 显示一句原文
- 用户点击麦克风朗读
- 浏览器 SpeechRecognition API 实时转文字
- 与原文对比，标注发音偏差
- 逐词打分（绿色=匹配，红色=不匹配/漏读）

### 4. 整篇影子跟读（Shadowing）
- 自动逐句播放原文 TTS
- 每句播放完后自动开启麦克风
- 用户跟读，记录语音
- 全文结束后出总分报告
- 可调速（0.5x / 0.75x / 1x / 1.25x）
- 播放模式：逐句（手动下一句）/ 连续（自动推进）

## 二、技术方案

### 技术栈
| 层 | 选型 | 理由 |
|----|------|------|
| 框架 | React 18 + TypeScript | 生态成熟，适合交互密集型应用 |
| 构建 | Vite | 快 |
| 样式 | Tailwind CSS | 轻量，开发快 |
| 语音识别 | Web Speech API (SpeechRecognition) | 浏览器原生，无需后端 |
| 语音合成 | Web Speech API (SpeechSynthesis) | 浏览器原生 TTS |
| 存储 | localStorage + IndexedDB (Dexie.js) | 离线可用，存文本和进度 |
| 字幕解析 | 自写 parser（srt/vtt 格式简单） | 体积小，无依赖 |
| 文本对比 | difflib 或自写 Levenshtein | 逐词对齐打分 |

### 浏览器兼容
- SpeechRecognition：Chrome/Edge 完整支持，Safari 部分支持，Firefox 不支持
- SpeechSynthesis：主流浏览器均支持
- 建议引导用户使用 Chrome

## 五、数据模型

```typescript
// 文本素材
interface TextMaterial {
  id: string
  title: string
  source: 'paste' | 'srt' | 'vtt' | 'txt'
  sentences: Sentence[]
  createdAt: number
  totalWords: number
}

interface Sentence {
  index: number
  text: string
  words: string[]        // 预分词
  startTime?: number     // 字幕时间戳（可选）
  endTime?: number
}

// 练习记录
interface PracticeRecord {
  id: string
  materialId: string
  mode: 'write' | 'speak' | 'shadow'
  results: SentenceResult[]
  overallScore: number
  duration: number
  createdAt: number
}

interface SentenceResult {
  sentenceIndex: number
  userInput: string
  originalText: string
  wordResults: WordResult[]
  accuracy: number       // 0-100
  attempts: number
}

interface WordResult {
  original: string
  user: string
  match: 'correct' | 'incorrect' | 'missing' | 'extra'
}
```

## 六、核心算法

### 文本对比（逐词对齐）
1. 原文拆词 → tokens[]
2. 用户输入拆词 → tokens[]
3. 用 Needleman-Wunsch 或简化 LCS 对齐
4. 标注每个词: correct / incorrect / missing / extra

### 语音对比（Speech-to-Text vs 原文）
1. SpeechRecognition 获取 transcript
2. transcript 与原文进行逐词对齐
3. 发音评分 = 匹配词数 / 总词数

### 影子跟读自动推进
1. TTS 播放当前句
2. 播放结束 → 自动开麦
3. 检测静音 2 秒 → 关闭麦克风
4. 对比评分 → 自动推进下一句
5. 循环直到全文结束

## 七、开发阶段

### Phase 1：基础框架（1天）
- Vite + React + TypeScript + Tailwind 项目搭建
- 路由配置（react-router）
- 基础 Layout 和页面骨架

### Phase 2：导入 + 解析（1天）
- 文本粘贴组件
- .srt/.vtt 字幕解析器
- 分句逻辑 + 文本入库（IndexedDB）
- 文本库列表页

### Phase 3：逐句书写模式（1-2天）
- 句子展示 + TTS 播放
- 输入框 + 实时对比
- 逐词高亮（正确/错误/遗漏）
- 进度追踪 + 分数

### Phase 4：逐句语音模式（1-2天）
- SpeechRecognition 集成
- 麦克风按钮 + 实时转写
- 语音转写 vs 原文逐词对比
- 发音反馈

### Phase 5：影子跟读模式（1-2天）
- 自动播放 + 自动录音循环
- 语速调节
- 连续/逐句模式切换
- 全文跟读报告

### Phase 6：打磨（1天）
- 练习报告页
- 数据持久化（进度记录）
- 浏览器兼容处理
- UI 细节优化

## 八、技术风险

| 风险 | 应对 |
|------|------|
| SpeechRecognition 只在 Chrome 好用 | 首页做浏览器检测，非 Chrome 弹提示 |
| 语音识别准确率不稳定 | 用模糊匹配而非精确匹配，降低挫败感 |
| 中文环境下英文识别可能差 | 设置 recognition.lang = 'en-US' |
| TTS 语音不够自然 | 可选：后续接 Azure/Google TTS API |
